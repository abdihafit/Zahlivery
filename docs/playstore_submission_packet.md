# Zahlivery Play Store Submission Packet

This is the exact set of files and assets you should have before uploading.

## Required project files

### Already prepared in the project
- Release signing support:
  - `android/app/build.gradle.kts`
- Signing template:
  - `android/key.properties.example`
- Upload checklist:
  - `docs/playstore_upload_checklist.md`
- Keystore creation guide:
  - `docs/create_upload_keystore.md`
- Store listing draft:
  - `docs/playstore_store_listing.md`
- Privacy policy template:
  - `docs/privacy_policy_template.md`
- Firebase Android config:
  - `android/app/google-services.json`
- App icon source:
  - `assets/images/zahlivery_icon.png`

## Files you must create locally

### Signing files
- `android/upload-keystore.jks`
- `android/key.properties`

### Release artifact
- `build/app/outputs/bundle/release/app-release.aab`

## External assets required by Play Console

These are not generated by Flutter and must be prepared separately:
- Privacy Policy URL
- Phone screenshots
- Feature graphic
- Support email
- App category choice
- Data safety responses
- Content rating responses

## Minimum upload path

1. Create the keystore
2. Create `android/key.properties`
3. Bump the version in `pubspec.yaml`
4. Run:

```powershell
flutter build appbundle --release
```

5. Upload:

```text
build/app/outputs/bundle/release/app-release.aab
```

## Package name currently configured

- `com.abdihafit.zahlivery`

## Required Firebase refresh after package rename

Before building the final release, register `com.abdihafit.zahlivery` as a new Android app inside Firebase for project `zahlivery-2a46d`, then replace:
- `android/app/google-services.json`
- `lib/firebase_options.dart`
