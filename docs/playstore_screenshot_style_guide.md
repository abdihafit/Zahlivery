# Zahlivery Play Store Screenshot Style Guide

Use this guide when designing the replacement `en-GB` screenshots for Google
Play. The goal is to make every image look consistent, readable, and accurate
to the actual app.

## Core rules
- Use real in-app screenshots as the base image.
- Keep the phone UI visible. Do not cover the whole screen with text.
- One message per screenshot.
- Only describe what is visible in that screenshot.
- Keep wording factual. Avoid hype words like `best`, `fastest`, `instant`,
  `live`, or `real-time`.
- Use the same brand colors and text placement across all screenshots.

## Recommended canvas
- Size: `1240 x 2208 px` portrait
- Safe margin: `64 px` on all sides
- Phone screenshot area: centered, about `820-900 px` wide
- Top text area: reserve the top `320-420 px`
- Bottom breathing room: keep at least `100 px` clear

If you are exporting at another Play-supported size, keep the same proportions.

## Layout formula for all screenshots
Every screenshot should follow this structure:

1. Background
- Use a very light solid or soft gradient background
- Recommended colors:
  - `#F5F8EF`
  - `#EEF6E6`
  - `#FFFDF7`

2. Header chip
- Small rounded label above the headline
- Examples:
  - `CUSTOMER`
  - `SHOP`
  - `RIDER`
  - `MULTI-ROLE`
- Style:
  - background: `#E7F3D7`
  - text color: `#2E5E00`
  - radius: `999 px`
  - font size: `28-34 px`
  - font weight: bold

3. Main headline
- Place near the top left or top center
- Max 2 lines
- Max 7 words per line when possible
- Font size: `72-92 px`
- Font weight: bold
- Text color: `#1D2A12`

4. Support line
- One short sentence below the headline
- Font size: `34-42 px`
- Text color: `#4B5B43`
- Keep this to one line if possible

5. Device screenshot
- Place the real app screenshot below the text block
- Add a subtle shadow
- Radius: `28-36 px`
- Do not tilt the phone unless the design still keeps the UI readable

## Suggested visual style
- Primary green: `#2E5E00`
- Accent green: `#6DBE00`
- Background light: `#F5F8EF`
- Text dark: `#1D2A12`
- Text muted: `#4B5B43`

## Typography guidance
- Use one clean bold font for headlines and one clean regular font for support
  text.
- Good options if available:
  - `Poppins`
  - `Montserrat`
  - `Nunito Sans`
  - `DM Sans`
- Do not mix many font families.
- Use sentence case, not all caps, for headlines.

## Simple composition template
For every screenshot, place elements in this order from top to bottom:
- Header chip
- Headline
- Support line
- Phone screenshot mockup

Use left alignment for all text unless the composition looks unbalanced.

## Export checklist
- Text is readable on a phone-sized preview
- No text overlaps the app UI
- Caption matches the screen content exactly
- No unsupported claims appear
- All six screenshots use the same style
- All six screenshots are in the same locale and spelling style

## Final warning
If a screenshot does not visibly show the feature named in the text, change the
text or replace the screenshot. Accuracy matters more than marketing flair for
this resubmission.
