# Zahlivery Play Store Upload Checklist

## Android package details
- Application ID: `com.abdihafit.zahlivery`
- Current version: `1.0.0+1`

## Files you need

### 1. Release signing files
- Create a keystore file:
  - `android/upload-keystore.jks`
- Create a real key properties file from the template:
  - copy `android/key.properties.example`
  - save it as `android/key.properties`

Required contents of `android/key.properties`:

```properties
storePassword=YOUR_STORE_PASSWORD
keyPassword=YOUR_KEY_PASSWORD
keyAlias=upload
storeFile=upload-keystore.jks
```

### 2. Play Store upload artifact
- Build and upload this file to Play Console:
  - `build/app/outputs/bundle/release/app-release.aab`

Preferred build command:

```powershell
flutter build appbundle --release
```

### 3. App icon source
- Current launcher icon source:
  - `assets/images/zahlivery_icon.png`

### 4. Firebase / app config files already in project
- `android/app/google-services.json`
- `lib/firebase_options.dart`

## Firebase follow-up after package rename

Because the Android package name has changed, you must also:
- add a new Android app in Firebase with package name `com.abdihafit.zahlivery`
- download the new `google-services.json`
- replace `android/app/google-services.json`
- regenerate `lib/firebase_options.dart` using FlutterFire or Firebase configuration for the new Android app

## Play Console items you must also prepare

These are not generated by Flutter and must be supplied in Play Console:

- App name
- Short description
- Full description
- Privacy Policy URL
- Screenshots for phone
- Feature graphic
- App category
- Contact email
- Data safety answers
- Content rating questionnaire

## Recommended release sequence

1. Create `android/upload-keystore.jks`
2. Create `android/key.properties`
3. Increase app version in `pubspec.yaml`
4. Run:

```powershell
flutter build appbundle --release
```

5. Upload:

```text
build/app/outputs/bundle/release/app-release.aab
```

## Important note

Right now the project will fall back to the debug signing key if `android/key.properties` does not exist.
That is useful for testing, but not acceptable for a proper Play Store release.
